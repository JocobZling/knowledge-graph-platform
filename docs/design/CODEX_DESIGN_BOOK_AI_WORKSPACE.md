# Codex Design Book：炫酷科技感 AI 工作台设计规范

> 目标：避免 Codex 生成“普通后台管理系统”或“AI 味很重但很廉价”的页面。  
> 设计方向：**未来感、空间感、光影感、数据感、可长期使用**。  
> 适用项目：个人知识图谱平台、AI 日报、智能测试日报、数据集管理、Agent 工作台、GitHub 自动化页面。

---

## 1. 总体设计定位

不要做成传统后台：

- 不要 Ant Design 默认风
- 不要 Element UI 默认风
- 不要大块纯蓝按钮
- 不要廉价渐变
- 不要满屏信息堆叠
- 不要“左侧菜单 + 表格 + 查询条件”的老式管理台

要做成：

> **AI OS / 个人智能控制中心 / 科技感知识工作台**

关键词：

- Dark futuristic
- Glassmorphism
- Aurora gradient
- Neural graph
- Floating cards
- Command center
- Cinematic dashboard
- Subtle motion
- High-end SaaS
- Data intelligence

## 4. 核心页面架构

### 4.1 首页 / AI Workspace Home

页面目标：第一眼就要像“智能控制中心”，不是后台系统。

结构：

```text
┌────────────────────────────────────────────────────┐
│ Top bar: Logo / Command Search / GitHub / Profile  │
├───────────────┬────────────────────────────────────┤
│ Sidebar       │ Hero: AI Workspace                 │
│ - Dashboard   │  - 今日摘要                         │
│ - AI Daily    │  - 当前 Agent 状态                   │
│ - Testing     │  - 知识图谱入口                      │
│ - Knowledge   │                                    │
│ - Dataset     │ Floating cards + Aurora background  │
│ - GitHub      │                                    │
└───────────────┴────────────────────────────────────┘
```

视觉要求：

- 背景使用深色渐变，不使用纯黑。
- 顶部有柔和极光背景。
- 中心区域使用悬浮玻璃卡片。
- 卡片边框使用 1px 半透明白色。
- 卡片 hover 时轻微上浮、边框发光。
- 首屏要有一个“AI Brain / Knowledge Graph / Orb / Data Stream”视觉焦点。

---

### 4.2 Dashboard

页面目标：数据清晰，但不能像传统报表。

模块建议：

- 今日 AI 简报
- 智能测试日报
- 最近 GitHub 更新
- 最近知识节点
- Agent 任务状态
- 数据集统计
- 重要待处理项

布局：

```text
左侧导航 + 顶部命令栏 + 12 栅格内容区

第一行：大 Hero 卡片 + 今日摘要
第二行：3 个状态卡片
第三行：数据趋势 + 最近更新
第四行：知识图谱预览 + Agent 活动流
```

---

### 4.3 知识图谱页

页面目标：像一个“认知网络”界面。

核心模块：

- 中央图谱画布
- 左侧节点分类
- 右侧节点详情
- 顶部搜索/过滤
- 底部时间线

视觉：

- 图谱节点使用发光圆点。
- 连接线使用低透明度线条。
- 选中节点出现 halo 光圈。
- 背景有 subtle grid / noise。
- 节点详情卡片使用 glass panel。

---

### 4.4 数据集管理页

页面目标：保留可用性，同时避免传统后台感。

建议设计：

- 数据集用卡片/列表混合视图，不要只做表格。
- 每个数据集卡片显示：名称、数据量、更新时间、状态、快捷操作。
- 表格编辑区可以采用更精致的 spreadsheet view。
- 筛选区域收敛到 command/search bar，不要大块查询表单。

---

### 4.5 日报页面

页面目标：像高质量科技 newsletter，而不是普通 Markdown 页面。

模块：

- 日期与标题
- 今日摘要
- 重点新闻
- Agent Ecosystem
- 智能测试相关
- GitHub 推送状态
- 参考来源

视觉：

- 内容卡片化
- 引用来源作为小标签
- 重要新闻使用 Spotlight Card
- 分类使用 chip / badge
- 支持 Markdown 渲染但外层要美化

```css
  line-height: 0.95;

```

```css

  font-size: 14px;
  line-height: 1.75;
```

原则：

- 英文标题可以更大、更锐。
- 中文正文保持清晰，不要过细。
- 标题和正文之间要有明显层级。
- 不要让所有文字都发光。

## 8. 动效规范

动效要“贵”，不要“闹”。

### 推荐

- Card hover：translateY(-4px)
- Button hover：轻微 scale
- 页面进入：fade + y: 12px
- 背景光晕：缓慢移动
- 图谱节点：微弱 pulse
- 数字统计：count up
- Markdown 内容：section reveal

### 不推荐

- 无限旋转大图标
- 满屏粒子乱飞
- 彩虹按钮
- 高频闪烁
- 每个组件都弹跳
- 过度 WebGL 导致卡顿

### Framer Motion 参考

```tsx
const fadeUp = {
  initial: { opacity: 0, y: 16 },
  animate: { opacity: 1, y: 0 },
  transition: { duration: 0.45, ease: "easeOut" }
}
```

## 9. 技术选型建议

如果是给 Codex 生成新前端，推荐：

```text
Next.js / Vite + React
Tailwind CSS
Framer Motion / Motion
Lucide React
React Flow 或 Sigma.js：知识图谱
Recharts / ECharts：数据图
MDX / react-markdown：日报渲染
shadcn/ui：基础组件骨架
React Bits / Magic UI / Aceternity UI：高级视觉组件
```

如果你当前项目仍以 Vue + Element UI 为主：

```text
可以保留业务页面逻辑，但新首页/工作台/日报展示页建议独立做一套更现代的 React 前端。
如果必须 Vue：
- Vue 3
- Tailwind CSS
- VueUse Motion
- Element Plus 只做基础表单，不暴露默认视觉
```

```text
你是一个高级前端设计工程师，不要生成传统后台管理系统风格。

请严格参考 docs/design/CODEX_DESIGN_BOOK.md 中的设计规范，为本项目生成一个具有未来科技感、AI OS 气质、玻璃拟态、暗色极光背景、悬浮卡片、克制动效的页面。

设计目标：
- 页面要有设计感，不要像 Element UI / Ant Design 默认后台。

- 视觉参考方向：Linear + Vercel + Stripe + Spline + React Bits + Awwwards WebGL。

- 保持中文内容的可读性。
- 页面首屏必须有明显视觉焦点。
- 动效克制，不要影响性能。
- 移动端可以简化，但桌面端要足够精致。

请先检查当前项目技术栈，再给出实施方案，随后修改代码。
```

## 11. 落地步骤

### 第一步：先做视觉骨架

目标：让首页第一眼变好看。

Codex 任务：

```text
读取 docs/design/CODEX_DESIGN_BOOK.md。
先不接真实接口，使用 mock data 实现 AI Workspace Home。
重点完成布局、背景、卡片、动效、响应式。
```

验收：

- 打开首页不再像传统后台。
- 背景、卡片、字体、间距统一。
- 不依赖真实接口也能展示完整视觉。

---

### 第二步：抽离 Design Tokens

Codex 任务：

```text
将颜色、圆角、阴影、字体、动效统一沉淀为 design tokens。
不要让颜色散落在各组件中。
```

验收：

```text
src/styles/tokens.css
tailwind.config.ts
src/components/ui/*
```

---

### 第三步：做 5 个核心复用组件

优先级：

1. AiCard
2. GlowButton
3. CommandSearch
4. MetricCard
5. SectionHeader

Codex 任务：

```text
请根据 docs/design/CODEX_DESIGN_BOOK.md 抽象 5 个可复用组件：
AiCard、GlowButton、CommandSearch、MetricCard、SectionHeader。
要求支持 className 扩展、暗色主题、hover 动效。
```

---

### 第四步：逐页替换

顺序：

1. 首页 / Dashboard
2. 日报页面
3. 知识图谱页面
4. 数据集页面
5. 设置页 / 登录页

不要一口气全部重构。

---

### 第五步：建立设计审查清单

每次 Codex 改页面后检查：

```text
是否像传统后台？如果是，重做。
是否有统一背景？如果没有，补。
是否卡片间距统一？如果没有，调。
是否默认蓝色太多？如果是，去掉。
是否动效过度？如果是，删。
是否移动端可用？如果不是，补。
是否组件可复用？如果不是，抽象。
```

## 12. Codex 负面约束

每次都加上：

```text
禁止：
- 生成 Ant Design / Element UI 默认后台风格
- 大面积使用 #409EFF / #1890FF
- 首屏只有表格和查询条件
- 使用廉价渐变
- 使用过多边框线
- 使用过多阴影
- 所有卡片都发光
- 所有元素都动
- 忽略中文排版
- 把颜色写死在每个组件里
- 生成不可维护的一次性页面
```

## 13. 推荐组件库清单

优先级：

1. shadcn/ui：基础组件骨架
2. React Bits：动画和高级视觉
3. Magic UI：Beam、Marquee、Border Beam、Sparkles
4. Aceternity UI：Hero、Background、Cards、Spotlight
5. 21st.dev：更有设计感的 React 组件和 AI Chat 组件
6. Framer Motion / Motion：统一动效
7. Lucide：图标
8. React Flow：知识图谱
9. Recharts / ECharts：数据图表
10. Spline：可选 3D 首屏视觉
